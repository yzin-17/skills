# Mockoon 随机长度与空数据设计

## 目标

让 `mockoon-gen` 生成的 Faker 数据覆盖更接近真实服务端边界的取值：未指定长度的字符串包含 `0` 到 `20` 的随机长度；整数覆盖负数、`0`、正数及 JavaScript 安全整数边界；并提供一项经用户确认后才启用的随机空数据模式，用于模拟服务端违反或未完整遵循 OpenAPI 契约的响应。

## 范围

- 集中调整成功响应模板的值渲染逻辑，覆盖 OpenAPI `format`、语义 Faker 映射与默认回退。
- 识别 OpenAPI 的字符串长度和数值范围约束，并在可表达时传给 Faker。
- 在初始化或从 OpenAPI 创建 artifact 的交互流程中，主动询问是否启用随机空数据模式。
- 将用户选择保存在 artifact 中，使重新渲染和导出结果可复现。
- 为空字符串、长度边界、整数边界及各类空态新增单元和端到端测试。

不在本次范围内：更改用户手写的 `bodyTemplate`、在 Mockoon 运行期动态修改配置、或为每个字段提供独立的空值概率。

## Artifact 配置

在 artifact 的 mock 配置中新增可选布尔字段 `randomEmptyData`：

```json
{
  "randomEmptyData": true
}
```

字段缺失或为 `false` 时保持契约型生成行为。创建 artifact 时 CLI 必须明确询问该选项，不能静默开启。`render-templates` 仅依据 artifact 中已保存的值重新生成模板，不再次提问。

## 值生成规则

### 正常模式

- 未指定长度的字符串使用随机长度 `0` 到 `20`；这包括 `string.sample` 和带语义映射但未声明长度的字符串生成器。
- 指定 `minLength` 或 `maxLength` 时，使用已声明边界；只指定一侧时，另一侧使用默认边界，并将结果规范为合法区间。
- 未指定范围的整数必须产生负数、`0` 与正数，并覆盖接近 `Number.MIN_SAFE_INTEGER` 和 `Number.MAX_SAFE_INTEGER` 的边界测试用例；指定 `minimum` / `maximum` 时优先遵守该范围。
- 枚举仍从声明值中选择，布尔值仍生成布尔值，数组与对象沿用现有结构生成规则。

### 随机空数据模式

启用 `randomEmptyData` 后，所有字段都有概率被替换为空态，不受 `required`、`nullable` 或其他 OpenAPI 约束限制：

- 字符串可以是空串 `""`；正常分支仍遵循长度规则。
- 数字、布尔、枚举及其他标量可以为 `null`。
- 数组可以为 `[]`，对象可以为 `{}`。
- 对象属性可以被省略，以表示 JSON 中无法表达的 `undefined` 与服务端未返回字段。

模板使用 Mockoon 支持的条件表达式实现随机分支。若某类值无法由 Mockoon 模板安全地表达省略，则保留 `null` 空态并在实现中记录限制，不能生成无效 JSON。

## 渲染与兼容性

现有字符串选择优先级不变：`format`、`semanticMappings`、`string.sample`。长度和空态作为所选 Faker 表达式外的参数或条件包装，不改变语义映射的决策方式。默认关闭空数据模式，已有 artifact 未升级也保持可用，重新渲染时获得新的随机长度策略。

## 测试

- 字符串默认长度可取 `0`、`1` 和 `20`，并验证已声明的长度边界优先。
- 语义映射与 `format` 的字符串同样接收长度策略。
- 整数覆盖负数、`0`、正数以及最小/最大安全整数边界；显式上下界保持优先。
- `randomEmptyData` 关闭时不出现额外空态；开启时覆盖 `null`、字段省略、空数组、空对象和空串。
- 验证生成模板始终是有效 JSON，并通过 CLI 端到端检查 artifact 的持久化、重新渲染和 Mockoon 导出。
